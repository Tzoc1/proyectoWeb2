🔧 getStatColor(value)
   Descripción: Retorna un color RGB basado en el valor del stat
   Parámetros:
      - value: Número del stat base (0-255)
   Retorna: Código de color HEX
      - Rojo (#FF6B6B) si value < 50

   Uso: Para colorear las barras de progreso en el modal

🔧 filterPokemons()
   Descripción: Filtra pokémons por tipo y/o generación
   Funcionalidad:
      1. Obtiene valores de los select de filtro
      2. Reinicia variables (offset, allPokemons)
      3. Si hay filtro de tipo: consulta API por tipo
      4. Si hay filtro de generación: consulta API por generación
      5. Si hay ambos: aplica ambos filtros
      6. Renderiza los resultados filtrados
   Variables utilizadas:
      - currentFilter: Almacena los filtros activos
      - allPokemons: Array de pokémons filtrados

───────────────────────────────────────────────────────────────────────────────

   Parámetros:
      - pokemons: Array de pokémons a filtrar

   Retorna: Array de pokémons que pertenecen a esa generación
   Uso: Se utiliza dentro de filterPokemons()

📄 BUSQUEDA.JS - MÓDULO DE BÚSQUEDA DE POKÉMON
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🔧 fetchpokemon(pokemon)
   Descripción: Obtiene datos completos de un pokémon desde la API
   Parámetros:
      - pokemon: Nombre o ID del pokémon (string/number)
   Funcionalidad:
      1. Realiza 2 consultas en paralelo:
         - API de pokémon base (stats, tipos, movimientos)
         - API de especies (generación, evoluciones)
      2. Si ambas consultas son exitosas: llama cambiarpokemon()
      3. Si hay error: muestra alerta "Pokémon no encontrado"
   Ventaja: Promise.all permite paralelizar las requests

───────────────────────────────────────────────────────────────────────────────

🔧 buscarpokemon()
   Descripción: Obtiene el valor del input y ejecuta la búsqueda
   Funcionalidad:
      1. Obtiene valor del input con ID 'buscar'
      2. Lo convierte a minúsculas y elimina espacios
      3. Si no está vacío: llama a fetchpokemon()

───────────────────────────────────────────────────────────────────────────────

🔧 cambiarpokemon(pokemon, species)
   Descripción: Actualiza el DOM con los datos del pokémon buscado
   Parámetros:
      - pokemon: Datos del pokémon (de API pokemon)
      - species: Datos de especies (para generación)
   Funcionalidad:
      1. Actualiza elementos del DOM:
         - Nombre del pokémon
         - Imagen (sprite frontal)
         - ID (#)
         - Tipo primario y secundario
         - Generación
         - Stats (HP, Attack, Defense, Sp. Atk, Sp. Def, Speed)
         - Altura (convertida a metros)
         - Peso (convertido a kg)
   Nota: Crea una vista formateada legible del pokémon

🔧 capitalize(str)
   Descripción: Convierte una cadena a mayúscula inicial
   Parámetros:
      - str: Cadena a convertir
   Retorna: String formateado
   Nota: Función duplicada (reutilizada en múltiples módulos)

🔧 loadMoves()
   Descripción: Carga movimientos con scroll infinito
   Funcionalidad:
      1. Verifica si ya está cargando (evita múltiples requests)
      2. Consulta API con limit=20 y offset actual
      3. Llama a renderMoves() con los resultados
      4. Incrementa offset para la próxima carga
   Offset: Comienza en 0, aumenta de 20 en 20
   Nota: Similar a loadData() de catalogo.js

───────────────────────────────────────────────────────────────────────────────

🔧 renderMoves(moves)
   Descripción: Renderiza las tarjetas de movimientos
   Parámetros:
      - moves: Array de movimientos desde API
   Funcionalidad para cada movimiento:
      1. Crea un div con clase 'move-card'
      2. Muestra:
         - Nombre del movimiento
         - Tipo (con color específico)
         - Potencia (power)
         - Precisión (accuracy)
         - Botón "Ver más"
      3. Agrega al contenedor (itemsContainer)
   Detalle: Usa Promise.all para obtener datos completos de cada movimiento

───────────────────────────────────────────────────────────────────────────────

🔧 showMoveModal(moveData)
   Descripción: Abre un modal con información completa del movimiento
   Parámetros:
      - moveData: Objeto movimiento desde API
   Contenido del Modal:
      1. Nombre del movimiento
      2. Tipo (con color)
      3. Tabla con información:
         - Potencia (poder del ataque)
         - Precisión (%)
         - PP (Points/Poder Político del movimiento)
         - Categoría (Physical/Special/Status)
         - Descripción del efecto
   Tabla: Usa clase 'teibol' para estilo

───────────────────────────────────────────────────────────────────────────────

🔧 Sentinel (Intersection Observer)
   Descripción: Detecta scroll al final para cargar más movimientos
   Funcionalidad:
      - Igual al sistema de catalogo.js
      - Ejecuta loadMoves() cuando se detecta

📄 DARKMODE.JS - MÓDULO DE TEMA OSCURO

🔧 enableDarkmode()
   Descripción: Activa el modo oscuro
   Funcionalidad:
      1. Agrega clase 'darkmode' al body
      2. Guarda 'active' en localStorage bajo clave 'darkmode'
   Persistencia: El tema se mantiene entre recargas

───────────────────────────────────────────────────────────────────────────────

🔧 disableDarkmode()
   Descripción: Desactiva el modo oscuro
   Funcionalidad:
      1. Remueve clase 'darkmode' del body
      2. Limpia localStorage (guarda 'null')

───────────────────────────────────────────────────────────────────────────────

🔧 Event Listener (theme-switch)
   Descripción: Alterna entre modo oscuro y claro
   Trigger: Click en elemento con ID 'theme-switch'
   Funcionalidad:
      1. Obtiene estado actual de localStorage
      2. Si NO está activo: llama enableDarkmode()
      3. Si está activo: llama disableDarkmode()
   Persistencia: El estado se guarda automáticamente

───────────────────────────────────────────────────────────────────────────────

🔧 DOMContentLoaded Initialization
   Descripción: Carga el tema guardado al iniciar
   Funcionalidad:
      - Si localStorage['darkmode'] === 'active': aplica enableDarkmode()
      - Asegura que el tema se mantenga entre sesiones
